GET e POST in PHP

Cosa sono GET e POST?

GET e POST sono i due metodi principali per inviare dati da un form HTML al server. PHP utilizza le variabili superglobali $_GET e $_POST per accedere a questi dati.

Metodo GET

Il metodo GET invia i dati attraverso l'URL. I parametri sono visibili nella barra degli indirizzi del browser.

<!-- Form HTML con metodo GET -->
<form action="elabora.php" method="GET">
    Nome: <input type="text" name="nome">
    Età: <input type="text" name="eta">
    <button type="submit">Invia</button>
</form>

<!-- File elabora.php -->
<?php
    // Recupera i dati dal form
    $nome = $_GET['nome'];
    $eta = $_GET['eta'];
    
    echo "Ciao " . $nome . ", hai " . $eta . " anni!";
    
    // URL risultante: elabora.php?nome=Mario&eta=25
?>

Caratteristiche del metodo GET:

  • I dati sono visibili nell'URL (es: pagina.php?nome=Mario&eta=25)
  • Limite di circa 2000 caratteri per l'URL
  • I dati possono essere salvati nei preferiti
  • NON usare per dati sensibili (password, informazioni personali)
  • Ideale per ricerche, filtri, paginazione

Metodo POST

Il metodo POST invia i dati nel corpo della richiesta HTTP. I dati non sono visibili nell'URL.

<!-- Form HTML con metodo POST -->
<form action="registrazione.php" method="POST">
    Username: <input type="text" name="username">
    Password: <input type="password" name="password">
    Email: <input type="email" name="email">
    <button type="submit">Registrati</button>
</form>

<!-- File registrazione.php -->
<?php
    // Recupera i dati dal form
    $username = $_POST['username'];
    $password = $_POST['password'];
    $email = $_POST['email'];
    
    echo "Registrazione completata per: " . $username;
?>

Caratteristiche del metodo POST:

  • I dati NON sono visibili nell'URL
  • Nessun limite pratico sulla quantità di dati
  • Più sicuro per dati sensibili
  • I dati non possono essere salvati nei preferiti
  • Ideale per form di registrazione, login, upload di file

Confronto GET vs POST

GET

  • Dati visibili nell'URL
  • Limite di lunghezza
  • Può essere salvato nei preferiti
  • Meno sicuro
  • Uso: ricerche, filtri

POST

  • Dati nascosti nell'URL
  • Nessun limite pratico
  • Non salvabile nei preferiti
  • Più sicuro
  • Uso: login, registrazione

Verificare se i Dati Esistono

È importante verificare se i dati sono stati inviati prima di utilizzarli:

<?php
    // Verifica con isset()
    if (isset($_POST['nome'])) {
        $nome = $_POST['nome'];
        echo "Ciao " . $nome;
    } else {
        echo "Nessun nome fornito";
    }
    
    // Verifica con empty()
    if (!empty($_GET['ricerca'])) {
        $ricerca = $_GET['ricerca'];
        echo "Stai cercando: " . $ricerca;
    }
    
    // Metodo alternativo (PHP 7+)
    $eta = $_POST['eta'] ?? 'Non specificata';
?>

Sicurezza: Validazione e Sanitizzazione

IMPORTANTE: Non fidarti mai dei dati ricevuti! Validali e puliscili sempre:

<?php
    // Pulire i dati
    $nome = htmlspecialchars($_POST['nome']);
    
    // Filtrare email
    $email = filter_var($_POST['email'], FILTER_SANITIZE_EMAIL);
    
    // Validare email
    if (filter_var($email, FILTER_VALIDATE_EMAIL)) {
        echo "Email valida!";
    }
    
    // Escape per database (con mysqli)
    $nome_sicuro = $connessione->real_escape_string($nome);
?>
